Hard   [Конфигурационный файл (config.xml)]
        1. Должен содержать версию менеджера, с которой может работать
        2. Добавлять простую информацию о компьютере, на котором создан файл
Easy   [Кнопки]
        1. "Загрузить новый модуль" - изменить на "добавить"
        2. "Открыть папку" - убрать
        3. "Очистить" - убрать (пока для нужна для тестов)
        4. "Копировать" - создать файл в этом же окне (заготовку из существующего)
Medium [Структура архива модуля]
        1. разбить распакованную папку data на две подпапки (головную папку назвать "unpack" ?)
            a) "data" - содержит файлы .jpg, .xml, .txt, .py
            b) "docs" - содержит файлы .pdf
        2. название всех картинок - "image.jpg"
        3. все названия файлов должны жестко зашиты в коде
        4. файлы "xml" и "py" должны быть закодированы sha
Hard   [Менеджер]
        1. Менеджер должен содержать таблицу совместимости с другими версиями менеджера
        2. Менеджер должен содержать таблицу с какими версиями МОДУЛЕЙ он может работать
        2. Менеджер должен содержать таблицу с какими версиями ПУЛЬТОВ он может работать
Hard   [Интерфейс]
        1. Окно вывода должно иметь скролл
        2. скрипт модуля перерисовывает окно менеджера
Hard   [Старт менеджера]
        1. Выбирает последний редактировавшийся (или использованный) модуль
        2. Восстанавливает параметры (настройки) менеджера, используемые в последний раз
Hard   [Менеджер]
        1. Создание системы совместимости версий (нужен класс-обертка над списком совместимых модулей с методами сравнения)
        2. Создание классов, хранящих своместимые модули и пульты
